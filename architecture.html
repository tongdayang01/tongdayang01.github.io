    ⚙️ 后端架构

    核心服务 · API路由 · 支付体系

    GenerateService(~5900+行) + PaymentService + 30+ API接口

    🧠 GenerateService — AI内容生成引擎（~5900+行）

    🌱 12+级优先级注入系统 (Layer -1 ~ Layer 12+)

    从系统级默认值 → 用户偏好 → 全局策略 → 模块配置 → 教育层次 → 专业适配 → 历史上下文 → 实时输入 → 动态覆盖，逐层叠加，确保每次生成精准适配当前场景。

    🎓 教育层次差异化

    中职 / 高职 / 职教本科 三个层次独立策略配置，同一模块输出不同深度的内容。中职偏重实操与工单，高职偏重方案设计，职教本科偏重理论体系与创新。

    ⚡ 异步生成 + 轮询机制

    POST /api/generate 提交任务 → 返回 taskId → 客户端轮询 GET /api/generate/result/:taskId → 获取完整结果。支持长文本、多模块并行生成。

    💳 PaymentService — 支付与解锁

    微信支付

    小程序端原生支持

    微信小程序内直接调起支付，支持模块解锁、会员购买、值餐下单、悦舍预约。

    抖音支付

    抖音小程序适配

    抖音小程序端的支付通道，与微信支付平行支持，覆盖更广用户群。

    模拟支付

    开发/演示环境

    开发环境和演示场景下的模拟支付，无需真实交易即可测试完整流程。

    Supabase订单存储

    订单持久化

    所有支付订单存储在Supabase数据库，支持订单查询、退款、统计分析。模块解锁状态本地+云端双写。

    📱 跨端兼容

    微信 · 抖音 · H5

    三端适配策略：

    • 微信小程序：原生组件（Camera/RecorderManager等）
    • 抖音小程序：抖音支付 + 抖音API适配
    • H5端：降级提示 + 兼容处理

    基于Taro跨端框架，一套代码三端运行

    💾 数据持久化

    localStorage + Supabase 双写

    • localStorage：本地缓存，离线可用
    • Supabase数据库：云端同步，多设备共享
    • 登录态管理：用户登录状态持久化
    • 已购模块缓存：本地记录解锁状态

    🎨 UI组件体系

    60+组件 · 原子化样式

    • shadcn/ui (Taro版)：60+可定制组件
    • Tailwind CSS 4：原子化样式引擎
    • lucide-react-taro：图标库
    • 深色/浅色主题：双主题支持
